Run a fast ideation sprint. Every idea worth keeping gets its own atomic note.

## Steps

1. Ask for the seed question or constraint (one question). Example prompts if I don't provide one:
   - *"What's a problem you want to think through?"*
   - *"What topic do you want to generate options for?"*
2. Generate 10–15 ideas rapidly. No filtering — volume first.
3. After the sprint, display all ideas numbered.
4. For each, I say: **keep**, **expand**, or **skip**.
   - **Keep**: create `00-Inbox/{{idea-slug}}.md` using `idea-note` template. Minimal — just title + one-line summary + raw text.
   - **Expand**: generate 5 sub-ideas or implications before saving.
   - **Skip**: discard.
5. After triage, report:
   - Ideas kept: N
   - Ideas expanded: N
   - Suggested clusters (ideas that group naturally → candidate project or MOC)
6. Ask: *"Want to run `/graduate` on any of these now, or let them sit in inbox?"*

Keep energy high. This is generative, not evaluative. Save the critique for `/challenge`.
